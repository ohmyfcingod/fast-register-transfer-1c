# Быстрый старт

Пять минут от `.cfe` до первого переноса. Подробности — в
[docs/УстановкаИЭксплуатация.md](docs/УстановкаИЭксплуатация.md).

## 1. Подключить расширение (обе базы)

Артефакт — [`build/БыстрыйПереносРегистров.cfe`](build/БыстрыйПереносРегистров.cfe).

1. Конфигуратор → *Конфигурация → Расширения конфигурации → Добавить* → выбрать `.cfe`.
2. **Снять** галки **«Безопасный режим»** и **«Защита от опасных действий»** → перезапустить.
   Без этого файловый I/O (`ЗначениеВФайл`) падает — это требование поставки, кодом не обходится.
   *(Без Конфигуратора: «Режим технического специалиста» → «Управление расширениями конфигурации» → выключить оба тумблера.)*

## 2. Первый перенос

Обработка **«Быстрый перенос регистров»** живёт в подсистеме **«Сервис»**.

| Шаг | Где | Действие |
|---|---|---|
| 1 | Источник | Вкладка **«Выгрузка»** → «…» у поля «Имена регистров» → отметить регистры → указать «Папку обмена» → **«Выгрузить»** |
| 2 | — | Перенести папку обмена на приёмник (или указать общую сетевую) |
| 3 | Приёмник | Вкладка **«Загрузка»** → та же папка → **«Загрузить»** (прогресс живьём, есть отмена) |

Формат по умолчанию — бинарный (`.bin`, быстрее всего). Для переноса между **версиями платформы**
переключите «Формат чанков» на **JSON**. Для узкой сети включите «Сжатие» = `9` (deflate, ~15% размера).
Профиль СУБД приёмника («Файловая ИБ» / «MS SQL Server» / …) сам подберёт размер чанка и число потоков.

## 3. Самопроверка (по желанию)

«Режим разработчика» = **Да** → вкладка **«Тесты»** → пресет **«сборочный smoke»** → **«Выполнить план»**.
Все строки должны стать `✅ OK`; сводный отчёт — на вкладке **«Отчёт»** и в `test_plan_report.html` папки обмена.
Пока идёт шаг, под вкладками моргает сова с кофе — это индикатор «сервер жив, а не завис».

Тесты гоняют синтетические регистры-фикстуры; **реальные регистры при этом не меняются**
(на них выполняется только выгрузка-чтение).

## 4. RabbitMQ (опционально)

Транспорт «папка → очередь → папка» на вкладке **«RabbitMQ»**. Нужен брокер и внешняя компонента
[PinkRabbitMQ](https://github.com/BITERP/PinkRabbitMQ) (`PinkRabbitMQ64.dll` в папку обмена — подберётся автопоиском):

```bash
docker run -d --name bpr-rabbit -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=rabbit -e RABBITMQ_DEFAULT_PASS=rabbit rabbitmq:3-management
```

Хост указывать **`127.0.0.1`**, не `localhost` (иначе Windows уводит в `::1`). Детали и грабли — в README.

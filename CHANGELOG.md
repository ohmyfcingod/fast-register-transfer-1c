# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — [SemVer](https://semver.org/lang/ru/).

## [1.0.0] — 2026-07-03

### Added
- **Панель тестов**: редактируемый план прогона (галка, объёмы, статус, время, вердикт) вместо россыпи кнопок; тесты разделены на **Сборочные** (корректность) и **Бенчмарки** (замеры); пресеты «сборочный smoke» / «перф файловая» / «перф MS SQL» (пресет применяет и профиль СУБД). Каталог тестов — один экспортный метод харнесса, план переживает закрытие формы (JSON в `ХранилищеОбщихНастроек`, ключ включает строку соединения ИБ).
- Сборочные тесты под каждую находку ревью v2: path traversal конверта RMQ, отмена и зомби-воркеры, сложные типы bin+json (UUID / ХранилищеЗначения / ссылки / составные), подчинённый регистр (замещение по регистратору).
- **Анимация-индикатор** («сервисный режим», Catppuccin): CSS-анимация в HTMLDocumentField живёт во время синхронного серверного вызова — visual heartbeat для человека и для агента со скриншотами; терминальная строка показывает текущий шаг.
- Сводный отчёт плана: таблица шагов + разворачиваемые секции, пишется инкрементально в `test_plan_report.html` после каждого шага.
- **RabbitMQ-транспорт** (PoC на PinkRabbitMQ): папка → очередь → папка, JSON-конверт, ack после записи файла, автопоиск компоненты в папке обмена и её родителе (папкой обмена часто выступает подпапка сеанса `Run_*`); если найден только ZIP — честная подсказка распаковать DLL (серверное подключение ZIP не берёт). Бинарь компоненты в поставку не входит.
- **Бенчмарк «Пропускная способность RMQ»**: синтетика → выгрузка → очередь → папка приёма; раздельный замер отправки и приёма (МБ/с), дренаж чужих сообщений перед стартом, сверка SHA-256 каждого файла после брокера.
- **JSON-формат чанков** (`СериализаторXDTO`, явное назначение типов) — портативность между версиями платформы; формат фиксируется в манифесте.
- **Фоновый перенос** с живым HTML-прогрессом (кролик), отменой и изоляцией прогонов подпапками сеанса `Run_*`.
- **Отбор по периоду** для РН/РБ/периодических РС; фиксируется в манифесте.
- Метрики OpenMetrics (`bpr_transfer.prom`) для Grafana / node_exporter.
- Профили производительности СУБД (файловая / MS SQL / PostgreSQL / экстремальный / вручную) с автоопределением.

### Fixed
- Зомби-воркеры отменённой выгрузки не подмешивают чанки в манифест следующего прогона (изоляция `Run_*`).
- Path traversal через имя файла в конверте очереди — жёсткая валидация при приёме.
- Запрещено смешивать в одной выгрузке с отбором периода периодические и непериодические регистры (дозапись давала бы дубли) — fail-fast до записи файлов.
- Гонка «Остановить» между фазами тика плана: стоп проверяется и перед серверным вызовом шага.
- Обрыв контекста сервера во время шага плана не оставляет вечную анимацию — цикл гасится штатно.

## [0.9.0] — 2026-06-27

### Added
- Движок бинарных чанков: `ЗначениеВФайл` + `manifest.json`, загрузка замещением по регистратору / дозаписью.
- Параллельная выгрузка и загрузка фоновыми заданиями (MIN/MAX-сплит диапазона; адаптивный запуск: БСП / собственный шим).
- Опциональное сжатие deflate (`ХранилищеЗначения`), докачка по sidecar-файлу, идемпотичные повторы.
- Нагрузочный харнесс: синтетический регистр, бенчмарки, `ПолныйТест` с HTML-отчётом.
- Упаковка в расширение конфигурации (`.cfe`, AddOn, 8.5.1).

### Fixed
- **Критический баг потери данных** в подчинённых регистрах (РН/РБ): граница чанка режется только на смене регистратора, движения одного регистратора не разрываются (находка внешнего ревью v1).
